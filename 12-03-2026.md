# Báo Cáo Triển Khai Xử Lý Lỗi (Bug/Defect/Failure) Trong Project
**Dự án:** Ứng dụng Quản lý Tài chính Cá nhân (Backend + Mobile App)
**Ngày thực hiện:** 12/03/2026

---

## 1. Bug / Logic Fault (Lỗi logic nghiêm trọng)

### 1.1 Lỗi trích xuất Tags/Mentions do Regex tiếng Việt (Frontend - Fix)
* **Vấn đề:** Hàm `NoteTagParser` (cho `#tag` và `@mention`) xài biểu thức chính quy (Regex) là `\w`. Biểu thức này không tự động nhận dạng các ký tự tiếng Việt (có dấu) trong bộ mã Unicode, dẫn đến hàm sẽ trả về kết quả rỗng (Null) hoặc cắt chữ tào lao tại nơi có dấu.
* **Cách fix:** Đã thay đổi pattern của Regex sang cấu trúc `RegExp(r'#([a-zA-Z0-9_À-ỹ]+)')`. Giờ đây app có thể đọc các hashtag/mention tiếng việt bình thường (Ví dụ: `#mua_sắm_đồ_cũ`).

### 1.2 Hiển thị số tiền gãy cấu trúc (Data Loss) (Frontend - Fix)
* **Vấn đề:** Hàm _formatAmount() mặc định luôn chia 1.000 và lược bỏ số tận cùng. Khi quét bằng máy ảnh biên lai mua hàng lẻ (Ví dụ 123.500 vnđ) và load giao diện, giá trị lại bị tròn khống làm thất thoát tiền trong Logic quản lý.
* **Cách fix:** Hiện tại, hàm đã đổi sang logic kiểm tra modulo chặt chẽ: chỉ làm tròn thành `'k'` hoặc `'tr'` nếu số tiền chính xác chia hết cho 1000 `% 1000 == 0`, nếu số lẻ lẻ sẽ giữ nguyên nguyên trạng `123500` không đổi.

### 1.3 Lỗi Parser dính phải dấu Phẩy (Backend - Fix)
* **Vấn đề:** Method `String.join(",", list)` kết dính mảng array List thành 1 String đơn lưu DB. Nếu hacker/user có ý truyền text chứa dấu phẩy vào tag, lúc lấy ra `split(",")` nó sẽ thành một đống các index rác, phá vỡ List.
* **Cách fix:** Viết vòng lặp `list.stream().map()` replace tất cả dấu ngầm định `,` bên trong text bằng kí hiệu gạch ngang `_` trước khi ghép mảng.

## 2. Defect (Khiếm khuyết trong Code/Thiết kế)

### 2.1 Không Validate Khóa ngoại Category (Backend - Fix)
* **Vấn đề:** Phương thức `create()` không hề chứng minh `categoryId` tồn tại hay không, nhét vô não `save()` luôn -> Hệ thống văng Exception chết server nếu API Tool truyền rác lên.
* **Cách fix:** Nâng cấp phương thức với vòng lặp check key ở Cache `idToNameMap()`. Nếu thiếu sẽ tự Raise `IllegalArgumentException` và kết nối với Class Controller để gửi mã `400 Bad Request` sạch sẽ.

### 2.2 Lỗi Tràn viền (Overflow) (Frontend - Fix)
* **Vấn đề:** UI hai nút bấm Đính kèm vị trí và Hình ảnh nằm trong thanh công cụ dùng Layout tĩnh (Row). Màn hình nhỏ sẽ làm tràn text ra khỏi thẻ container dẫn đến lỗi vạch vàng (Overflow).
* **Cách fix:** Chuyển cấu trúc Container `Row` thành hệ sinh thái Widget `Wrap(spacing: 8)`.

## 3. Failure / Missing Feature (Tính năng không hoạt động/Thiếu tính năng cốt lõi)

### 3.1 Upload Hình Ảnh hoàn toàn Fake (Backend/Frontend - Fix)
* **Vấn đề:** Flutter chỉ cắm link hình cục bộ, không gửi API. Backend không hề có code nhận stream file Data (MultiPart-Form) ghi vào trong Server. File không đồng bộ.
* **Cách fix:** 
  - **Backend:** Thêm hàm `uploadImage()` sinh động đường dẫn URL vật lý (File Transfer bằng UUID). Xây thêm `@Configuration WebMvcConfig` dùng ResourceHandler để phơi tĩnh thư mục public `/uploads/**` chạy ra Internet.
  - **Frontend:** Code bổ sung Service API hàm `apiClient.uploadImage()`. Tích hợp xử lý Async trong file App sau khi user Insert DB thành công, tự động send gói Image đính kèm theo form-data.

### 3.2 Hổng Cấu Trúc Update (Frontend/Backend - Fix)
* **Vấn đề:** User bấm tạo và không bao giờ Undo sửa sai được. Không hề có Route/Logic/Button Update cho toàn hệ thống App.
* **Cách fix:** 
   - Backend viết thêm Route `@PutMapping("/{id}")` và override biến entity set fields.
   - Flutter viết thêm tính năng `onTap` nhận `entryToEdit` parameter truyền qua `showModalBottomSheet()` -> form `AddEntryModal` lấy dữ liệu ra đính lên giao diện thông minh. Gọi nút Lưu API Update thay vì Add.

## 4. Performance Anomaly / Problem (Vấn đề về Hiệu năng)

### 4.1 Truy vấn dư thừa n+1 (Backend - Fix)
* **Vấn đề:** Category API bị gọi Query `SELECT *` liên tục mỗi khi Load danh sách List Transaction hoặc lướt app, do `CategoryService` không có cache tĩnh. Quá tải Database và giật Lag server kinh hãi.
* **Cách fix:** Nâng cấu trúc thành Singleton `idToNameMapCache` có cấu trúc Cache đồng bộ Thread (synchronized). Tái sử dụng vùng nhờ Memory để lấy HashMap Category ngay tức khắc với Speed `O(1)`.

## 5. Inconsistency (Sự thiếu nhất quán)

### 5.1 Rủi ro Bảo Mật & Validation Server (Backend - Fix)
* **Vấn đề:** Bỏ quên `@Size` Validation và set `@NotNull` cho Object Ngày giờ trong `CreateEntryRequest.java`.
* **Cách fix:** Gắn mác giới hạn mảng Array list tối đa `< 20 thẻ Tag/Mention` chặn cuộc tấn công DDos, cắn đứt Validation cứng ngắc trên backend.
